# Installing Road Trip Weather on Android & iOS

The app is fully client-side (no server needed), so on both platforms there
are two ways to get it onto a phone: install the PWA straight from the
browser, or build a native package with Capacitor.

## First: host the app

Both PWA paths need the static bundle on an HTTPS host (GitHub Pages,
Netlify, Cloudflare Pages, …):

```bash
python build_www.py     # produces the www/ folder
# upload www/ to your static host
```

(For quick LAN testing you can run `python app.py` and open the machine's
address from the phone, but browsers only offer installation over HTTPS or
localhost.)

## Android

### Option 1 — Install as a PWA (no build tools needed)

1. Open the hosted URL in **Chrome** on the phone.
2. Menu (⋮) → **Add to Home screen** → **Install**.

It launches fullscreen with its own launcher icon, and the app shell works
offline (live forecasts still need a connection).

### Option 2 — Build an APK with Capacitor

Needs [Node.js](https://nodejs.org) and
[Android Studio](https://developer.android.com/studio) (any OS). From the
project folder:

```bash
npm install               # Capacitor CLI + platforms
npm run android:add       # one-time: builds www/ and creates the android/ project
npm run android:sync      # after any code change
npm run android:open      # opens Android Studio
```

In Android Studio: **Build → Build APK(s)**, then sideload
`android/app/build/outputs/apk/debug/app-debug.apk`, or **Generate Signed App
Bundle** for the Play Store. With a phone on USB debugging,
`npm run android:run` launches directly on the device.

## iOS

### Option 1 — Install as a PWA (no build tools needed)

1. Open the hosted URL in **Safari** on the iPhone/iPad (must be Safari —
   other iOS browsers can't install PWAs).
2. Tap the **Share** button (the square with an arrow) → **Add to Home
   Screen** → **Add**.

It gets its own home-screen icon (the 180px `apple-touch-icon`) and launches
standalone without Safari UI. iOS quirks to know:

- There is no install banner like Chrome's — users must find *Add to Home
  Screen* in the Share sheet themselves.
- Offline shell caching works (service workers are supported since iOS 11.3),
  but iOS may evict the cache after a few weeks of disuse.

### Option 2 — Build a native iOS app with Capacitor

**Requires a Mac** with [Xcode](https://developer.apple.com/xcode/) and
[Node.js](https://nodejs.org) — Apple's toolchain doesn't run on Windows or
Linux. (Free Apple ID: install on your own device for 7 days at a time; paid
Apple Developer account: TestFlight / App Store.) From the project folder on
the Mac:

```bash
npm install
npm run ios:add           # one-time: builds www/ and creates the ios/ project
npm run ios:sync          # after any code change
npm run ios:open          # opens Xcode
```

In Xcode: select your team under **Signing & Capabilities**, pick your device,
and **Run** (▶). For distribution use **Product → Archive**.

## Notes

- App ID is `com.roadtripweather.app` (change it in `capacitor.config.json`
  before publishing).
- Icons are generated by `python make_icons.py` into `static/icons/`;
  Capacitor uses `www/` as its web root (see `build_www.py`).
- The app calls Nominatim, OSRM's demo server, and Open-Meteo directly. These
  are free public services with usage policies — fine for personal use, but
  host your own OSRM / use a commercial router before distributing widely.
